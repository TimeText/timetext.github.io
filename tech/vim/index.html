Use

  • 翻页:
    • 翻一页：ctrl-f 和 ctrl-b，f=forword b=backward
    • 翻半页：ctrl-d 和 ctlr-u ，d=down u=up
    • 翻一行 ctrl-e 和 ctrl-y

Get

  • 非root权限vim编辑后强制保存：:w !sudo tee %

  • 键盘映射：map <F3> :命令<CR> （F3和:之间有空格）

  • 插入模式中不能删除换行问题： set backspace=indent,eol,start

  • 批量注释：先按v再选行，Ctrl + v进入块模式，输入注释符号，连按两次Esc；

    • 取消注释：先按v再选行，Ctrl + v进入块模式，选择注释符号，按d删除。

  • vim打开是doc格式，报错：Not an editor command: ^M 替换“^M”提示：Pattern not found: ^M

    • 解决方法：vim 将文件从dos格式转换到unix格式，编辑vim时候按：进入命令模式，输入 set fileformat=unix，然后保存重新进入。

分屏

  1. 打开文件并且分屏 vim -o[n] file1 [file2 ...] vim -O[n] file1 [file2 ...] 小写o是水平分屏,大写O是垂直分屏;中括号表示可有可无,n分屏的个数
  2. 在vim中的命令模式下分屏
    • 垂直分屏：
      • 命令：:vs [file2] 后面不跟文件名是将当前文件垂直分屏;(也可用 vsp或者vsplit代替vs)
      • 快捷键：ctrl+w + v (同时按下ctrl和w键后,松开再按v键:快捷键操作只可将当前文件分屏)
    • 水平分屏
      • 命令： :sv [file2] 后面不跟文件名是将当前文件水平分屏;跟文件名是将新文件在水平分屏中打开(也可用 sp或者split代替sv)
      • 快捷键：ctrl+w + s (同时按下ctrl和w键后,松开再按s键:快捷键操作只可将当前文件分屏)
    • 新建文件水平分屏
      • 命令：:new [newfile] 如果跟有文件名则会新建一个文件,当写入内容并且保存退出后会新建该文件(如果没有写入内容保存退出也不会新建该文件)；
      • 快捷键：ctrl+w + n (同时按下ctrl和w键后,松开再按n键:如果没有跟文件名,在写入内容后保存时命令模式下输入::w filename会新建一个名字为filename的文件
  3. 移动光标
    • 把光标移到右边的屏中：ctrl+w l
    • 把光标移到左边的屏中 ：ctrl+w h
    • 把光标移到上边的屏中：ctrl+w k
    • 把光标移到下边的屏中：ctrl+w j
    • 把光标移到下一个屏中：ctrl+w w
    • 把光标移到上一个屏中：ctrl+w p
  4. 移动分屏
    • 向右移动：ctrl+w L
    • 向左移动：ctrl+w H
    • 向上移动：ctrl+w K
    • 向下移动：ctrl+w J
    • 向下旋转窗口：ctrl+w r
    • 向上旋转窗口：ctrl+w R
    • 当前窗口与下一个窗口对调：ctrl+w x
  5. 控制分屏的尺寸
    • 减少当前窗口高度（对上下分屏才有用） ctrl+w - 按完ctrl w 后松开,同时按下shift -
    • 增加当前窗口高度（对上下分屏才有用） ctrl+w + 按完ctrl w 后松开,同时按下shift +
    • 增加当前窗口宽度（对左右分屏才有用） ctrl+w > 按完ctrl w 后松开,同时按下shift >
    • 减少当前窗口宽度（对左右分屏才有用） ctrl+w < 按完ctrl w 后松开,同时按下shift <
    • 使所有窗口恢复均等 ctrl+w =
  6. 关闭分屏：
    • 关闭除当前分屏外的其他分屏
      • 命令：:only
      • 快捷键:ctrl+w o (最后一个分屏不能用此快捷键关闭)
    • 关闭当前分屏
      • 快捷键:ctrl+w c (最后一个分屏不能用此快捷键关闭)
    • 关闭当前分屏
      • 快捷键:ctrl+w q (可用来关闭最后一个分屏)
    • 关闭所有分屏
      • 命令::qa

""定义函数SetTitle，自动插入文件头 
func SetTitle() 
    "如果文件类型为.sh文件 
    if &filetype == 'sh' 
        call setline(1,"\#########################################################################") 
        call append(line("."), "\# File Name: ".expand("%")) 
        call append(line(".")+1, "\# Author: ") 
        call append(line(".")+2, "\# mail: ") 
        call append(line(".")+3, "\# Created Time: ".strftime("%c")) 
        call append(line(".")+4, "\#########################################################################") 
        call append(line(".")+5, "\#!/bin/bash") 
        call append(line(".")+6, "") 
    else 
        call setline(1, "/*************************************************************************") 
        call append(line("."), "    > File Name: ".expand("%")) 
        call append(line(".")+1, "    > Author: ") 
        call append(line(".")+2, "    > Mail:  ") 
        call append(line(".")+3, "    > Created Time: ".strftime("%c")) 
        call append(line(".")+4, " ************************************************************************/") 
        call append(line(".")+5, "")
    endif
    if &filetype == 'cpp'
        call append(line(".")+6, "#include<iostream>")
        call append(line(".")+7, "using namespace std;")
        call append(line(".")+8, "")
    endif
    if &filetype == 'c'
        call append(line(".")+6, "#include<stdio.h>")
        call append(line(".")+7, "")
    endif
    "新建文件后，自动定位到文件末尾
    autocmd BufNewFile * normal G
endfunc

配置CPP补全

  1. 下载clangd：https://github.com/clangd/clangd/releases/tag/11.0.0
  2. 解压clangd：unzip clangd-linux-11.0.0.zip
  3. 将解压后的bin目录加入到系统环境变量中，或者将其中的clangd软连接到/usr/bin/clangd：sudo ln -sf ~/Downloads/clangd_11.0.0/bin/clangd /usr/bin/clangd
  4. 安装coc-clangd，在vim中输入以下命令即可安装它：:CocInstall coc-clangd
  5. 打开C++文件时就会自动补全

airblade/vim-gitgutter

激活

您可以显式关闭和打开vim-gitgutter（默认为on）：

  • 与关闭 :GitGutterDisable
  • 用打开 :GitGutterEnable
  • 用切换:GitGutterToggle。

要为每个缓冲区切换vim-gitgutter：

  • 与关闭 :GitGutterBufferDisable
  • 用打开 :GitGutterBufferEnable
  • 切换 :GitGutterBufferToggle

您可以打开和关闭标志（默认为打开）：

  • 用打开 :GitGutterSignsEnable
  • 与关闭 :GitGutterSignsDisable
  • 用切换:GitGutterSignsToggle。

您可以打开和关闭行高亮显示（默认为关闭）：

  • 用打开 :GitGutterLineHighlightsEnable
  • 与关闭 :GitGutterLineHighlightsDisable
  • 用切换:GitGutterLineHighlightsToggle。

请注意，如果打开并突出显示了行，则您将有一个空的符号列-更准确地说，是带有不可见符号的符号列。这是因为行高亮显示需要符号，并且即使有符号可见，Vim / NeoVim总是在有符号时显示符号列。

使用Neovim 0.3.2或更高版本，您可以打开和关闭行号高亮显示（默认为关闭）：

  • 用打开 :GitGutterLineNrHighlightsEnable
  • 与关闭 :GitGutterLineNrHighlightsDisable
  • 用切换:GitGutterLineNrHighlightsToggle。

相同的警告适用于行号突出显示和上方的行突出显示。

如果您同时关闭了行高亮显示和标志，则看不到标志列。

preservim/nerdcommenter

使用

  • <leader> 表示斜杠\

  • [count]<leader>cc ：注释从当前行往下数的 count 行，count 可省略，默认值为 1 (cc 是 NERDCommenterComment 的缩写)。

  • [count]<leader>cu：取消注释从当前行往下数的 count 行，count 可省略，默认值为 1(cu 是 NERDCommenterUncomment 的缩写)。

  • [count]<leader>ci：切换从当前行往下数的 count 行的注释状态(ci 是 NERDCommenterInvert 的缩写)。

  • <leader>c$：注释掉从当前光标所在位置到行尾的剩下内容，等价于在光标当前所在位置插入注释符 ( c$ 表示 NERDCommenterToEOL 的缩写)。

  • <leader>cA：在光标所在当前行行尾添加注释符号并进入 Vim 的插入模式 ( cA 是 NERDCommenterAppend 的缩写)。

  • <leader>c<space> 加上/解开注释, 智能判断

  • <leader>cy 先复制, 再注解(p可以进行黏贴)

iamcco/markdown-preview.vim

如果需要预览数学公式，还需要安装 mathjax-support-for-mkdp 插件：

Plug 'iamcco/mathjax-support-for-mkdp'
Plug 'iamcco/markdown-preview.vim'

使用和设置

命令：

    MarkdownPreview
    " 在打开 markdown 文件后，使用该命令可以打开预览窗口

    MarkdownPreviewStop
    " 关闭 markdown 预览窗口，并停止开启的服务进程

在 MarkdownPreview 命令无效的情况下，可以先 MarkdownPreviewStop 再 MarkdownPreview

默认配置：

    let g:mkdp_path_to_chrome = ""
    " 设置 chrome 浏览器的路径（或是启动 chrome（或其他现代浏览器）的命令）
    " 如果设置了该参数, g:mkdp_browserfunc 将被忽略

    let g:mkdp_browserfunc = 'MKDP_browserfunc_default'
    " vim 回调函数, 参数为要打开的 url

    let g:mkdp_auto_start = 0
    " 设置为 1 可以在打开 markdown 文件的时候自动打开浏览器预览，只在打开
    " markdown 文件的时候打开一次

    let g:mkdp_auto_open = 0
    " 设置为 1 在编辑 markdown 的时候检查预览窗口是否已经打开，否则自动打开预
    " 览窗口

    let g:mkdp_auto_close = 1
    " 在切换 buffer 的时候自动关闭预览窗口，设置为 0 则在切换 buffer 的时候不
    " 自动关闭预览窗口

    let g:mkdp_refresh_slow = 0
    " 设置为 1 则只有在保存文件，或退出插入模式的时候更新预览，默认为 0，实时
    " 更新预览

    let g:mkdp_command_for_global = 0
    " 设置为 1 则所有文件都可以使用 MarkdownPreview 进行预览，默认只有 markdown
    " 文件可以使用改命令

    let g:mkdp_open_to_the_world = 0
    " 设置为 1, 在使用的网络中的其他计算机也能访问预览页面
    " 默认只监听本地（127.0.0.1），其他计算机不能访问

按键

按键绑定例子（F8打开预览窗口，F9关闭预览窗口）：

nmap <silent> <F8> <Plug>MarkdownPreview        " 普通模式
imap <silent> <F8> <Plug>MarkdownPreview        " 插入模式
nmap <silent> <F9> <Plug>StopMarkdownPreview    " 普通模式
imap <silent> <F9> <Plug>StopMarkdownPreview    " 插入模式

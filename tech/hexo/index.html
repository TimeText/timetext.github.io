初始化

Hexo更多命令：

hexo clean					#清理缓存文件

hexo s 						#运行本地服务器
hexo d						#上传到服务器
hexo new "name"	 			 #新建文章
hexo new page "pagename"	  #新建页面
hexo help 					 #查看帮助
hexo version 				 #查看版本信息
hexo deploy -g 				 #生成加部署
hexo server -g 				 #生成加预览

部署到Gitee

  1. Gitee新建仓库

  2. 配置SSH

    本地生成：ssh-keygen -t rsa -C "yourname@gitee.com"

    查看公钥：cat ~/.ssh/id_rsa.pub

    公钥配置到Gitee

    测试链接：ssh -T git@gitee.com

    返回结果：Hi xxxx! You've successfully authenticated, but GITEE.COM does not provide shell access.

  3. 在_config.yml中配置Git

    deploy:
type: git
repo: https://gitee.com/用户名/仓库名.git
branch: master

  4. 安装自动部署发布工具：npm install hexo-deployer-git --save

  5. 三连发布博客：hexo clean && hexo g && hexo d,首次发布需要在shell中输入账号(邮箱)和密码。

  6. 仓库-服务-Gitee Pages-选择master分支-开启HTTPS-更新

hexo转typecho

本文摘自：hexo-2-typecho

项目地址：https://github.com/jiangmitiao/hexo-2-typecho

作者博客：https://blog.gavinzh.com/

转换原理：使用插件将本地md转换成typecho数据库格式

安装

npm install --save hexo-2-typecho

配置

在_config.yml中增加以下配置，修改对应值即可

typecho:
	authorId: 1
	images: 'images'
	mid: 10
	cid: 10

默认情况:

typecho的authorId为1

typecho的cid和mid均为10

hexo图片的默认保存位置是source文件下的images文件夹中

修改配置：

如果你的typecho有多个用户的情况下，你可以查看typecho数据库，选出一个作者id。

如果你的typecho已经写了不少博客，你需要查看typecho中cid和mid最大值是多少，然后将配置的cid和mid分别+1。

使用hexo-admin写作图片一般保存在images文件夹中，如果你的图片存在别的文件夹下，则需要修改。

生成

调用hexo g命令，会在目录下生成typecho文件夹。文件夹中的文件如下：

调用hexo g命令，会在目录下生成typecho文件夹。文件夹中的文件如下：

typecho
├── contents.json
├── metas.json
├── relation.json
└── usr
   └── uploads
       └── hexo

其中contents.json对应typecho的contents表，metas.json对应typecho的metas表，relation.json对应typecho的relationships表。

usr文件夹中存储的是转移好的图片。

迁移

使用生成的usr文件夹覆盖typecho中的usr文件夹

使用Navicat将json数据导入到typecho中对应的表，当然你也可以使用别的软件，但是我还是建议你使用Navicat。

Github自动部署Hexo

利用GitHub Actions 实现自动部署 Hexo。两个仓库，一个仓库hexo-md存放源文件（可以设置成私有），一个仓库hexo-html存放前端，修改源文件后自动生成部署到html仓库。

步骤

  • 本地可以通过 hexo clean && hexo g && hexo d 生成运行上传hexo，在 .github/workflows/deploy.yml新建一个监听代码，然后上传到hexo-md仓库，当监听到仓库变化时候自动生成部署到hexo-html仓库。为了两个仓库的安全，需要在生成一对公钥私钥，私钥放到源文件仓库hexo-md中，公钥放到前端代码仓库hexo-html中。

  • 在Hexo配置文件中添加前端仓库地址：

    deploy:
  type: git
  repository: git@github.com:GithubName/GithubName.Github.io.git
  branch: master

  • 生成密钥对：ssh-keygen -t rsa -b 4096 -C "Hexo Deploy Key" -f github-deploy-key -N ""，生成两个文件：github-deploy-key （密钥）和 github-deploy-key.pub （公钥）

    ssh-keygen -t rsa -b 4096 -C "Hexo Deploy Key" -f github-deploy-key -N ""

参数说明：
	-b	指定密钥长度。对于RSA密钥，最小要求768位，默认是2048位。DSA密钥必须恰好是1024位(FIPS 186-2 标准的要求)。
	-C	提供一个新注释
	-f	指定密钥文件名。
	-N	提供一个新的密语。
	-t	指定要创建的密钥类型。可以使用："rsa1"(SSH-1) "rsa"(SSH-2) "dsa"(SSH-2)

  • 在Github的两个仓库中配置密钥：

    • 存放源文件的hexo-md仓库配置私钥：Settings -> Secrets->New secret
      • 名称Name填写HEXO_DEPLOY_KEY（和下面GitHub Actions Workflow中的一致），
      • 内容Value是github-deploy-key 中的内容。
    • 存放前端代码的hexo-html仓库配置公钥：Settings -> Deploy keys -> Add deploy key
      • Title中输入HEXO_DEPLOY_PUB或者其他名称，
      • Key 中粘贴 github-deploy-key.pub 文件的内容。
    • 一定要勾选 Allow write access 来打开写权限。

  • 创建Workflow：在本地中新建 .github/workflows/deploy.yml，代码为：

    name: Hexo Deploy

on:
  push:
    branches:
      - master

jobs:
  build:
    runs-on: ubuntu-latest
    if: github.event.repository.owner.id == github.event.sender.id

    steps:
      - name: Checkout source
        uses: actions/checkout@v2
        with:
          ref: master

      - name: Setup Node.js
        uses: actions/setup-node@v1
        with:
          node-version: '12'

      - name: Setup Hexo
        env:
          ACTION_DEPLOY_KEY: ${{ secrets.HEXO_DEPLOY_KEY }}
        run: |
          mkdir -p ~/.ssh/
          echo "$ACTION_DEPLOY_KEY" > ~/.ssh/id_rsa
          chmod 700 ~/.ssh
          chmod 600 ~/.ssh/id_rsa
          ssh-keyscan github.com >> ~/.ssh/known_hosts
          git config --global user.email "Actions@auto.com"
          git config --global user.name "Actions"
          npm install hexo-cli -g
          npm install

      - name: Deploy
        run: |
          hexo clean
          hexo deploy

  • 将本地代码push到存放源文件的hexo-md仓库就会自动部署上传到hexo-html仓库。

教程总结

  1. 本地安装python3和requests包

    • 检测python是否安装：本地终端输入python 或者 python3无报错且可以启动python

    • 检测requests是否安装：本地终端输入pip show requests 或者 pip3 show requests无报错且输出request相关信息。

    • 若request未安装：pip install requests 或者 pip3 install requests

    • 若pip或pip3未安装/未找到：python pip 安装与使用【链接直达】

    • 若python3未安装：python3环境搭建【链接直达】

  2. 网线连接电脑和路由器使二者同一局域网，本地连接地址修改成192.168.31.88

  3. 运行本地web服务器

    • 打开chfs-windows-x64-1.0文件夹中的chfs.exe（仅windows 64位支持）

    • 若为其他系统可以终端使用python3 -m http.server 在chfs-windows-x64-1.0目录下运行python自带的web服务器。

  4. 进入telnet

    • 打开路由器管理页面192.168.31.1登录后在网页地址栏获取stok的值，

    • 打开OpenWRTInvasion-0.0.3文件夹并运行一个终端，

    • 使用python remote_command_execution_vulnerability.py

    • 或者python3 remote_command_execution_vulnerability.py运行脚本，

      看到终端输出“are u ok”时后输入以下命令

      cd /tmp
wget http://192.168.31.88/shared/breed-mt7621-pbr-m1.bin
mtd -r write breed-mt7621-pbr-m1.bin Bootloader

      输入完成后路由器会重启，路由器上的指示灯光会变暗，请不要担心，属正常现象。

  5. 将本地连接地址获取方式改为自动

  6. 进入breed

    • 网线连接路由器和电脑，浏览器输入192.168.1.1进入breed。

    • 若提示无法找到页面等进不去breed：拔下路由器电源，长按RESET按钮并插入电源，刷新网页知道进入breed再松手。

  7. 在breed内备份eeprom

    eeprom解释：其中包含路由器mac地址等信息，若mac地址与硬件不匹配可能会影响信号网速等。

    备份方法：

    • [固件备份]==>选择[EEPROM备份]、[编程器备份]等保存到本地；
    • 找到[MAC地址修改]==>截图保存MAC地址；
    • 关于其他选项卡也建议截图备份一下；
    • 截图或者下载压缩包到本地后建议再创建2~3份副本。

  8. 在breed内刷入固件

    • [固件备份]选项==>选择[常规固件]==>选择你在网上下载的或者自己编译的[固件]，未完不待续。
      • 若是第一次刷入固件：[EEPROM]选项==>选择你刚刚备份到本地的EEPROM备份；
    • 勾选自动重启；
    • 点击[上传]按钮，会让你再确认一次，确认即可。
    • 上传完成后会自动重启。

  9. 连接路由器

    • 重启后找到WIFI名为编译固件时定义的名称，默认为openwrt，默认无连接密码；
    • 连接后浏览器输入编译固件时定义的地址，默认192.168.1.1，账号root，密码password；<-密码时password不包括这个分号；

  10. 进行配置

    • 设置WiFi名称，WiFi密码，各类插件等..........

通过breed间接刷写固件教程

间接刷写的意思是不能在breed界面直接刷入固件，

例如我手里的小米R4A千兆版路由器，在openwrt官网下载编译的固件通过breed直接刷写后无限重启进不去openwrt系统，若要使breed直接刷入的系统可以正常使用，需要在编译时进行参数修改。

参考教程

[R4A]终于解决了breed刷机后不能正常启动的问题 【链接直达】

教程总结

  1. 原因分析

    1. breed的作用是引导系统，即从某一个地址开始时系统，地址的分区称为[闪存布局]；
    2. 并没有R4A专属的breed，之前刷的breed是从其他地方搬过来给R4A用的，会有异常；
    3. R4A和官方原版openwrt固件中的启动地址均为0x160000，但是breed并不知道，故循环重启；
    4. 解决方法：告诉breed从0x160000这个地址开始启动系统。
    5. 关于直刷：直刷固件是在固件编译时修改相关参数实现可以通过breed直接刷入。

  2. 准备阶段

    1. 路由器进入breed并网线连接电脑，教程参上；
    2. 进入固件目录并打开终端输入python3 -m http.server 启动本地web服务器；
    3. 获取本机和路由器同网段的ip地址，即应为192.168.1.X，通常为192.168.1.2；

  3. 刷入阶段

    1. telnet 192.168.1.1：打开终端使用telnet连接breed；
    2. wget http://本地web服务器/固件资源名称：从本地web服务器下载固件到路由器；
    3. flash erase 0x160000 0x微大于固件大小的地址：为新固件腾出空间；
    4. flash write 0x160000 0x固件存储地址 0x微大于固件大小的地址：刷写固件数据到内存中：
    5. boot flash 0x160000：在0x160000地址块重启系统；
    6. 恭喜您：重启进入新系统，断电重启仍然无限重启，因为breed还不知要在0x160000地址块重启系统。

  4. 完善阶段

    1. 重启进入breed
    2. 设置环境变量
      • 字段=autoboot.command
      • 值= boot flash 0x160000
    3. 保存、断电、重启
      • 若成功进入系统，恭喜您；
      • 若仍然无限重启，同情你；

  5. 示例

    因手里机器暂不可复现，仅提供部分代码

    Starting breed built-in shell

breed> wget http://192.168.1.2:8000/openwrt-21.02.3-ramips-mt7621-xiaomi_mi-router-4a-gigabit-squashfs-sysupgrade.bin
Connecting to 192.168.1.2:8000... connected.
HTTP request sent, awaiting response... 200 OK
Length: 5767792/0x580270 (5MB) []
Saving to address 0x80001000

[========================================================================] 100%

Transmission completed in 1.9s.

    其中：

    Length: 5767792/0x580270 (5MB) []:是所下载固件的大小，即固件大小所占的地址

    Saving to address 0x80001000:是固件保存到了哪里，保存到了0x80001000这个地址开头的，往后推固件大小个地址就是保存的固件地址区域了。

教程总结

  • 使用breed刷入编程器固件以恢复官方系统时候 只勾选自动重启；

  • 默认密码：提供的all.bin刷入后第一次启动是已经配置好的，WIFI连接密码和后台管理密码可能都是8个1：11111111，上次忘记验证了，可能是8个1，如果不正确，请长按REST按钮恢复出厂设置。

  • 异常：修改后在breed刷入后，重启进入官方ROM，但在管理页面点击[马上体验]按钮提示“Invalid nonce”和“not auth”

    • 解决：长按REST按钮恢复出厂设置，解决。

  • 异常：重启重置进入官方配置页面后，MAC地址正确但是SN号不正确。

    • 解决：telnet连接后进行修改，连接方式见上文。

      bdata set CountryCode=CN     #设置地区代码
bdata set SN=XXXXXX/XXXXXXX   #修改路由器SN
bdata sync && bdata commit   #保存修改

#再次依次输入
nvram set SN=XXXXX/XXXXXXXX   #修改路由器SN
nvram commit   #保存修改

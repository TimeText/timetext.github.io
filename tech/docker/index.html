虚拟机容器Docker的学习笔记

理论知识

  • 什么是docker：是基于 Go 语言实现的开源项目，提供高效、敏捷、轻量级的容器方案，支持在多种云平台和本地系统上部署。Docker 引擎的基础是 Linux 容器（Linux Containers，LXC）技术。

  • docker对比虚拟机：

    • Docker 的启动和停止可以在秒级实现，比传统的虚拟机方式要快得多。
    • Docker 容器对系统资源的需求很少，一台主机上可以同时运行上千个 Docker 容器。
    • Docker 通过类似 Git 的操作来获取、分发和更新应用镜像，指令简明，学习成本较低。
    • Docker 通过 Docker file 配置文件来支持灵活的自动化创建和部署机制，提高工作效率。

  • docker镜像：Docker 镜像是一个只读的 Docker 容器模板，含有启动 Docker 容器所需的文件系统结构及其内容。Docker 镜像是 Docker 容器的静态视角，Docker 容器是 Docker 镜像的运行状态。

  • docker仓库Repository：集中存放镜像文件的场所。分为公开/私有仓库。

  • 注册服务器Registry：存放着多个仓库，每个仓库集中存放某一类镜像，使用不同的标签（Tag）来进行区分。目前最大的公开 Registry 是 Docker Hub（https://hub.docker.com）

  • docker容器Container：从镜像创建的相互隔离的应用的运行实例。镜像是只读的。容器从镜像启动时，Docker 会在镜像的最上层创建一个可写层，镜像本身保持不变。

命令

安装

  • 安装：yum -y install docker
  • 启动：systemctl start docker
  • 开机自启：systemctl enable docker

镜像

  • 搜索镜像：docker search [OPTIONS] TERM

  • 拉取镜像：docker pull [OPTIONS] NAME[:TAG|@DIGEST]

    • 默认镜像仓库：Docker Hub 公共注册服务器中的仓库
    • 不显式地指定 Tag，则默认会下载 latest 标签，即仓库中最新版本的镜像

  • 查看本地镜像：docker images

    • IMAGE ID 唯一标识了镜像

  • 查看镜像详细信息：docker inspect [OPTIONS] NAME|ID [NAME|ID...]

  • 使用镜像创建一个新的容器并运行一个命令：docker run [OPTIONS] IMAGE [COMMAND] [ARG...]

    • OPTIONS
      • -d：后台运行容器，并返回容器ID
      • -i：以交互模式运行容器，通常与 -t 同时使用
      • -t：为容器重新分配一个伪输入终端，通常与 -i 同时使用

  • 将指定镜像保存成 tar 归档文件：docker save [OPTIONS] IMAGE [IMAGE...]

    • OPTIONS：-o 输出到指定文件名称

  • 导入使用docker save命令导出的镜像：docker load [OPTIONS]

    • OPTIONS：
      • --input , -i : 指定导入的文件，代替 STDIN；
      • --quiet , -q : 精简输出信息；

  • 删除本地一个或多少镜像：docker rmi [OPTIONS] IMAGE [IMAGE...]

    • OPTIONS：
      • -f :强制删除；
      • --no-prune :不移除该镜像的过程镜像，默认移除；

容器

  • 创建一个新的容器并运行一个命令：docker run [OPTIONS] IMAGE [COMMAND] [ARG...]

    • OPTIONS
      • -d：后台运行容器，并返回容器ID
      • -i：以交互模式运行容器，通常与 -t 同时使用
      • -t：为容器重新分配一个伪输入终端，通常与 -i 同时使用

  • 创建一个新的容器但不启动它：docker create [OPTIONS] IMAGE [COMMAND] [ARG...]

    • 语法同docker run

  • 列出容器：docker ps [OPTIONS]

  • OPTIONS

    • -a :显示所有的容器，包括未运行的。
    • -l :显示最近创建的容器。

  • 容器启动/停止/重启：docker [start/stop/restart]

  • 查看容器详细信息：docker inspect [OPTIONS] NAME|ID [NAME|ID...]

  • 获取容器的日志：docker logs [OPTIONS] CONTAINER

    • OPTIONS
      • -f :跟踪日志输出
      • -t :显示时间戳
      • --tail :仅列出最新N条容器日志

  • 连接到正在运行中的容器：docker attach [OPTIONS] CONTAINER

  • 查看容器中运行的进程信息：docker top [OPTIONS] CONTAINER [ps OPTIONS]

  • 在运行的容器中执行命令：docker exec [OPTIONS] CONTAINER COMMAND [ARG...]

    • OPTIONS
      • -d：分离模式: 在后台运行
      • -i：即使没有附加也保持STDIN 打开
      • -t：分配一个伪终端

  • 删除一个或多个容器：docker rm [OPTIONS] CONTAINER [CONTAINER...]

    • OPTIONS
      • -f：通过 SIGKILL 信号强制删除一个运行中的容器。
      • -l：移除容器间的网络连接，而非容器本身。
      • -v：删除与容器关联的卷。

  • 导出容器，将文件系统作为一个tar归档文件导出到STDOUT：docker export [OPTIONS] CONTAINER

    • OPTIONS：-o 输出到指定文件名称

  • 导入容器，从归档文件中创建镜像：docker import [OPTIONS] file|URL|- [REPOSITORY[:TAG]]

    • OPTIONS
      • -c：应用docker 指令创建镜像；
      • -m：提交时的说明文字；

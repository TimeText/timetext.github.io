解决Github设置邮箱隐私后无法push的问题

 2022.7.5  734

在github个人设置中开启【Keep my email addresses private】和【Block command line pushes that expose my email】后，push时候提示“error: failed to push some refs to 'https://github.com/xxxx/xxxx.git'”的解决方法。

点此展开网页英语原文

[x]Keep my email addresses private

We’ll remove your public profile email and use xxxxxxxxx+uesename@users.noreply.github.com when performing web-based Git operations (e.g. edits and merges) and sending email on your behalf. If you want command line Git operations to use your private email you must set your email in Git.

Commits pushed to GitHub using this email will still be associated with your account.

[x]Block command line pushes that expose my email

When you push to GitHub, we’ll check the most recent commit. If the author email on that commit is a private email on your GitHub account, we will block the push and warn you about exposing your private email.

[x]将我的电子邮件地址保密

在执行基于 Web 的 Git 操作（例如编辑和合并）并代表您发送电子邮件时，我们将删除您的公开个人资料电子邮件并使用 xxxxxxxxx+uesename@users.noreply.github.com。如果您希望命令行 Git 操作使用您的私人电子邮件，您必须在 Git 中设置您的电子邮件。

使用此电子邮件推送到 GitHub 的提交仍将与您的帐户相关联。

[x]阻止暴露我的电子邮件的命令行推送

当您推送到 GitHub 时，我们将检查最近的提交。如果该提交的作者电子邮件是您 GitHub 帐户上的私人电子邮件，我们将阻止推送并警告您不要暴露您的私人电子邮件。

解决方法1：关于该选项后使用原配置进行push；

解决方法2：将本地配置的邮箱修改成分配的隐私邮箱并刷新后再次push，隐私邮箱为你在页面看到的形如“xxxx+uesename@users.noreply”的邮箱；

点此展开解决方法2
# 查看当前的全局用户email
git config --global user.email

# 重新设置你的全局用户email,形如“xxxx+uesename@users.noreply”的邮箱
git config --global user.email "xxxx+uesename@users.noreply"

# 重置作者信息
git commit --amend --reset-author

# 再次push
git push

### 查看与修改用户名同理
git config --global user.name

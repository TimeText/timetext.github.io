最终意图：自定义指令bk，bk new实现新建博客；bk push实现推送到仓库.....等操作。

流程概览：编写本地sh脚本用于完成各种操作；指令别名用于实现在任何位置执行脚本；

Mac 终端设置指令别名

由于Mac终端使用时是zsh，所有需要在zsh的配置文件中引入bash的配置文件

~/.zshrc中引入~/.bash_profile

# ~/.zshrc

# 当有 ~/.bash_profile 文件时，才加载这个文件
test -f ~/.bash_profile && source ~/.bash_profile

~/.bash_profile中输入别名设置

# ~/.bash_profile

alias blog="xxxxx/xxxx/xxxx/auto_hugo.sh"

编辑sh脚本

auto_hugo.sh所在位置应同~/.bash_profile中的一样

#!/bin/bash 

echo "hello"

进行别名测试

任意位置打开终端输入blog回车，输出hello，则表示设置正常。

完善sh脚本

我的sh脚本如下，位置在博客根目录下。但因我的hugo目录配置有较大diy改动，所以新增博文和修改博文部分请按需所取，关于git的push部分应该是不需要改动就可以使用的，放在这里也主要是为了进行备份。

#!/bin/bash

# 配置博客根目录
rootdir="/xxxxx/hugo-blog/"

#================================
#================================

deftest(){
	echo "测试功能"
}

recerror(){
	echo ""
	echo "===================="
	echo "|error|error|error|error|error|"
	echo "错误:${1}"
	echo "|error|error|error|error|error|"
	echo "===================="
	echo ""
	exit 9
}
defrun(){
	cd ${rootdir}
	hugo server -D &
	open 'http://localhost:1313'
	
}
defadd(){
		cd ${rootdir}
		echo "===================="
		cd ./content/zh-cn/ && ls -d */ && cd ${rootdir}
		echo "===================="
		echo -n "请选择博文类别(回车默认为tech):"
		read categories
		if [ -z  ${categories} ];then
			categories="tech"
		fi
		if [ ! -d "./content/zh-cn/${categories}" ];then
			recerror "类别输入错误，我被你气死了，若你感到自责就在上面选择正确分类并重启我。"
		fi
		echo -n "请输入博文标题:"
		read title
		if [ -z ${title} ];then
			recerror "你好像并没有输入什么东西，我已经饿死了，若你感到自责请重启我"
		else
			hugo new ${categories}/${title}.md
			mv ./content/zh-cn/${categories}/${title}.md ./content/zh-cn/${categories}/$(date "+%y%m%d")-${title}.md
		fi
		sleep 1s
		vim ./content/zh-cn/${categories}/$(date "+%y%m%d")-${title}.md
}

defedit(){
		cd ${rootdir}
		echo "===================="
		cd ./content/ && ls -d */ && cd ${rootdir}
		echo "===================="
		echo -n "请选择博文语言(回车默认为zh-cn):"
		read language
		if [ -z  ${language} ];then
			language="zh-cn"
		fi
		if [ ! -d "./content/${language}/" ];then
			recerror "类别输入错误，我被你气死了，若你感到自责就在上面选择正确分类并重启我。"
		fi
		echo "===================="
		cd ./content/${language}/ && ls -d */ && cd ${rootdir}
		echo "===================="
		echo -n "请选择博文类别(回车默认为tech):"
		read categories
		if [ -z  ${categories} ];then
			categories="tech"
		fi
		if [ ! -d "./content/${language}/${categories}" ];then
			recerror "类别输入错误，我被你气死了，若你感到自责就在上面选择正确分类并重启我。"
		fi
		fpath=$(dirname "$0")
		echo ""
		echo "*****************"
		echo  "cd  ${fpath}/content/${language}/${categories} && ls" 
		echo "*****************"
		echo "：）复制粘贴以上、按回车键「Enter」"
		echo "*****************"
}
defsave(){
	cd ${rootdir}
	git add .
	echo "===================="
	echo -n "请输入commit(回车默认为“$(date "+%y%m%d-%H%M%S")-"update"”):"
	read categories
	if [ -z  ${categories} ];then
		categories=$(date "+%y%m%d-%H%M%S")-"update"
	fi
	git commit -m ${categories}

}
defpush(){
	cd ${rootdir}
	git push
}
#================================
#================================

if [ $# == 0 ];then
	echo "=======帮助信息help======"
	echo "run:本地运行-Run now"
	echo "add:写新博文-Creat New Blog"
	echo "edit:改旧博文-Edit Old Blog"
	echo "save:保存到本地-Add and Commit"
	echo "push:推送到远程-Push to Git"
	echo "========================="
	echo "前往博客根目录请使用："
	echo ""
	echo "cd  " ${rootdir}
	echo ""
fi

if [ $# -gt 1 ];then
	recerror "输入这么多参数，你想撑死我吗？感到自责就重启我"
fi

if [ $# == 1 ];then
	cd ${rootdir}
	if [ $1 == "run" ]; then
		defrun
	elif [ $1 == "add" ]; then
		defadd
	elif [ $1 == "edit" ];then
		defedit
	elif [ $1 == "save" ];then
		defsave
	elif [ $1 == "push" ];then
		defpush
	elif [ $1 == "test" ];then
		deftest
	else
		recerror "输入这么多参数，你想撑死我吗？感到自责就重启我"
	fi
fi

延伸：解决sh脚本内不能跳转到其他目录

意图：通过tob或者其他自定义命令前往blog的主目录或其他目录内。

问题：在sh脚本内编写cd dir，执行sh脚本不能跳转。

原因：shell在执行sh脚本时候会fork一个子进程，子进程通过exec执行程序。

参考：Linux下通过Shell脚本快速进入指定目录，文章的方案是使用source执行脚本，例如source goto.sh，但和我的意图还是有所距离，一个字母都不想多输入。

解决：使用给source goto.sh加别名的方式解决

例如：

~/.bash_profile


alias tob="source /home/user/to.sh"

/home/user/to.sh

#!/bin/bash

cd /bin/

完成后使用 source ~/.bash_profile刷新。

测试：任意位置打开终端，输入tob回车后当前目录会跳转到/bin目录中。
